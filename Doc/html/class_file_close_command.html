Detailed Description

Command that handles closing the currently loaded file/catalog.

This command resets the current catalog, effectively "closing" the session, and provides user feedback through the console output writer.

void FileCloseCommand::execute ( const std::vector< std::string > & params)
overridevirtual

Executes the file close operation.

Executes the close command, resetting the loaded catalog state.

Parameters
paramsParameters passed to the command (ignored in this case).

This function performs the necessary actions to "close" the currently loaded database catalog. It first verifies that a catalog is indeed open by checking the context.loadedCatalogExists flag. If a catalog is open, it prints a confirmation message to the console, then proceeds to clear the contents of the context.loadedCatalog (by assigning an empty Catalog object) and updates the context.loadedCatalogExists flag to false.

Parameters
paramsA constant reference to a vector of strings containing command-line arguments. For the close command, these parameters are not used, but the signature is required by the Command::execute interface.
Exceptions
std::runtime_errorif no catalog is currently loaded (context.loadedCatalogExists is false). The exception message will guide the user to open a file first.

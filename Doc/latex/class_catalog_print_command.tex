\doxysubsection{Detailed Description}
\doxylink{class_command}{Command} to display the content of a specified table with pagination. 

This command allows users to view the data stored within a table loaded in the current database catalog. It prints the table\textquotesingle{}s header (name, column names, and types) followed by its rows, presented in a paginated format for easy viewing of large datasets.

Executes the \textquotesingle{}print\textquotesingle{} command, displaying a table\textquotesingle{}s data with pagination. 

Executes the \textquotesingle{}print\textquotesingle{} command, displaying the content of a table with pagination.

This method retrieves the specified table from the loaded catalog. It then prints the table\textquotesingle{}s overall name, followed by its column names and their respective data types as headers. Subsequently, all rows of the table are fetched as strings and passed to {\ttfamily \doxylink{class_i_o_utils_ad6f428a52d903d0ff47010209200519f}{IOUtils\+::print\+In\+Page\+Format}} for paginated output to the console. This allows users to browse large tables page by page.

This function processes the command-\/line arguments to obtain the target table name. It performs validation checks to ensure a catalog is loaded and the correct number of parameters (2) is provided.

The method then retrieves the specified table from the {\ttfamily context.\+loaded\+Catalog}. It first prints the table\textquotesingle{}s name. Following that, it iterates through the table\textquotesingle{}s columns to print all column names on one line, and then all column types on the next line, serving as a header for the data.

All rows of the table are then retrieved as formatted strings using {\ttfamily table.\+get\+Row\+As\+String(i)}. These rows are collected into a {\ttfamily std\+::vector\texorpdfstring{$<$}{<}std\+::string\texorpdfstring{$>$}{>}}. Finally, {\ttfamily \doxylink{class_i_o_utils_ad6f428a52d903d0ff47010209200519f}{IOUtils\+::print\+In\+Page\+Format}} is used to output these rows to the console with pagination, allowing the user to navigate through large result sets. Any errors (e.\+g., table not found) are caught and reported to the console.

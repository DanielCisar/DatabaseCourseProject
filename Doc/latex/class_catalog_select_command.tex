\doxysubsection{Detailed Description}
\doxylink{class_command}{Command} to select and display rows from a table that meet a specific condition. 

This command allows users to filter and view data from a table based on a value match in a designated column. All rows satisfying the condition are displayed in a paginated format.

Executes the \textquotesingle{}select\textquotesingle{} command, filtering and printing matching table rows. 

Executes the \textquotesingle{}select\textquotesingle{} command, displaying rows matching a column condition.

This method identifies rows in a specified {\ttfamily table\+Name} where the value in {\ttfamily search\+Column\+Index} matches {\ttfamily search\+Value}. It retrieves the table and the target column. For each row, if the condition is met, the entire row is formatted as a string and collected. The collected matching rows are then displayed to the console using a paginated format via {\ttfamily \doxylink{class_i_o_utils_ad6f428a52d903d0ff47010209200519f}{IOUtils\+::print\+In\+Page\+Format}}.

This function processes the command-\/line arguments to obtain the column index for searching, the value to match, and the target table name. It performs validation checks to ensure a catalog is loaded, parameters are correct, and the table exists.

The method then retrieves the specified table from the {\ttfamily context.\+loaded\+Catalog} and the column to search within. It iterates through all rows of the table. For each row, it checks if the value in the {\ttfamily column\+To\+Search} matches the {\ttfamily value} argument. If a match is found, the entire row is retrieved as a formatted string using {\ttfamily table.\+get\+Row\+As\+String(i)} and added to a {\ttfamily std\+::vector\texorpdfstring{$<$}{<}std\+::string\texorpdfstring{$>$}{>}}.

Finally, the collected matching rows are passed to {\ttfamily \doxylink{class_i_o_utils_ad6f428a52d903d0ff47010209200519f}{IOUtils\+::print\+In\+Page\+Format}} for paginated output to the console, allowing the user to browse filtered data. Any errors (e.\+g., table not found, invalid column index) are caught and reported.

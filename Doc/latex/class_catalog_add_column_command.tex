\doxysubsection{Detailed Description}
\doxylink{class_command}{Command} to add a new column to an existing table in the loaded catalog. 

This command facilitates schema modification by allowing users to extend an existing table with a new column. It requires the target table\textquotesingle{}s name, the desired name for the new column, and its data type. The newly added column will be initialized with NULL (empty) values for all existing rows.

Executes the add column command, modifying the specified table. 

Executes the \textquotesingle{}addcolumn\textquotesingle{} command, adding a new column to a specified table.

This method adds a new column to the table identified by {\ttfamily params\mbox{[}1\mbox{]}}. The new column will have the name specified in {\ttfamily params\mbox{[}2\mbox{]}} and the type (String, Integer, or Double) specified in {\ttfamily params\mbox{[}3\mbox{]}}. For all existing rows in the table, the new column will be populated with NULL values.

This function processes the command-\/line arguments to identify the target table, the name for the new column, and its data type. It performs validation checks\+:
\begin{DoxyEnumerate}
\item Ensures a catalog is currently loaded.
\item Verifies the correct number of parameters (4) is provided.
\item Validates that the specified column type string is one of "{}\+String"{}, "{}\+Integer"{}, or "{}\+Double"{}.
\end{DoxyEnumerate}

Upon successful validation, a new {\ttfamily \doxylink{class_table_column}{Table\+Column}} of the specified type and name is created using {\ttfamily \doxylink{class_column_factory}{Column\+Factory}}. This new column is then pre-\/filled with NULL values for all existing rows in the table. Finally, the new column is added to the table. Status messages are printed to the console upon success or error.
